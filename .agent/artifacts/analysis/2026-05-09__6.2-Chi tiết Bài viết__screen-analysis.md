## 7) API Endpoints
| Method | Path | Auth | Request | Response | Error codes |
|--------|------|------|---------|----------|-------------|
| GET | `/api/v1/blog/{slug}` | 🌐 | - | `BlogPost` object | 404: Not found |
| POST | `/api/v1/contacts` | 🌐 | `{ email, subject: "newsletter", ... }` | `{ success: true }` | 422: Invalid email |

> [!NOTE]
> Theo API list, hiện chưa có endpoint riêng cho Newsletter, có thể dùng tạm `/contacts` với subject đặc biệt hoặc [ASSUMPTION] chờ BE cung cấp endpoint mới.

## 8) Business Rules
- **BR-01 (View Count)**: Mỗi lần truy cập bài viết chi tiết, hệ thống phải tự động tăng `view_count` lên 1 (xử lý phía Backend khi gọi API chi tiết).
- **BR-02 (TOC Auto-generate)**: Mục lục bài viết được trích xuất tự động từ các thẻ `h2`, `h3` trong nội dung bài viết.
- **BR-03 (Reading Progress)**: Thanh tiến trình đọc (Reading Progress Bar) phải phản ánh chính xác vị trí cuộn trang của người dùng so với toàn bộ nội dung bài viết.

## 10) Edge Cases
- **EC-01**: Nội dung bài viết cực dài (>10,000 từ) gây lag khi parse TOC.
- **EC-02**: Bài viết không có bất kỳ thẻ `h2`, `h3` nào -> Ẩn section Table of Contents.
- **EC-03**: Tác giả bài viết bị xóa -> Hiển thị "Anonymous" hoặc fallback UI.

## 11) Assumptions & Open Questions
### Assumptions
- [ASSUMPTION] A-01: Newsletter sẽ được tích hợp vào hệ thống Contact hiện tại của dự án.
- [ASSUMPTION] A-02: Toàn bộ ảnh trong bài viết hỗ trợ Lazy Loading để tối ưu performance.
