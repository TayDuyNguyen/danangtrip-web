# Core Agents

This repository intentionally keeps a small local agent set.
These profiles exist to support the project-specific `.agent/skills/` workflow, not to mirror a generic marketplace catalog.

## Active Agents

- `orchestrator.md`
- `project-planner.md`
- `explorer-agent.md`
- `frontend-specialist.md`
- `backend-specialist.md`
- `test-engineer.md`
- `security-auditor.md`
- `devops-engineer.md`
- `debugger.md`

## Rules

- Prefer local skills first; use agents as perspective guides.
- Do not reintroduce removed generic roles unless the repository actually needs them.
- If a new agent is added later, update this file and the platform adapters together.
